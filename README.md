**Альтернатива консольным утилитам. Аппаратно-ускоренный GUI для обхода ограничений DPI.**

Ускорение Telegram Desktop, YouTube, Discord и полная кастомизация.

> **ZAPRET CORE** — это аппаратно-ускоренный графический пользовательский интерфейс (GUI) и система оркестрации для локального обхода систем глубокого анализа пакетов (DPI).
>
> В отличие от классических VPN-сетей, эта утилита не маршрутизирует трафик через сторонние удаленные серверы. Внедрение, фрагментация и модификация пакетов происходят локально на уровне ядра ОС. Такой подход позволяет обходить ограничения провайдера, сохраняя 100% пропускной способности канала и исходный сетевой пинг.

> ### РЕАКЦИЯ АНТИВИРУСОВ И СИСТЕМЫ
> Программа использует драйвер `WinDivert64.sys` для обработки сети на уровне ядра ОС. Некоторые антивирусы склонны классифицировать файлы WinDivert как высокорискованные. Обнаружение всегда будет называться `WinDivert` или `Not-a-virus:RiskTool.Multi.WinDivert`.
>
> Это абсолютно нормальное поведение. WinDivert — это легитимный инструмент для перехвата трафика. Для стабильной работы добавьте папку с утилитой в исключения вашего антивируса.

## ZAPRET CORE: ОБНОВЛЕНИЕ v2.4.3

Итерация полностью посвящена интерфейсу. Setup Node для мобильных переписана с нуля и стала отдельной полноценной страницей с пошаговыми гайдами под каждую платформу. Раздел LOGS приведён к строгой консольной форме без декоративных элементов. Подвкладка «Сервисы» в SETTINGS убрана — навигация снова сжата до Профили / Тест / Инструменты.

### Setup Node 2.0 (мобильная страница на :8081)

- **Шесть пошаговых вкладок** вместо трёх: **iOS / Android / Windows / macOS / Browser / Manifest**. Активная вкладка автоматически подбирается по `User-Agent` — iPhone сразу видит инструкцию для iOS, Android — свою, и так далее.
- **Два QR-кода** теперь рядом: PAC-конфиг и сырой MTProto host:port. Любой увеличивается тапом — открывается оверлей-модалка.
- **`tg://proxy?...` deep-link** — кнопка под QR-кодами, которую сервер заполняет твоим IP, портом и секретом. На устройствах с установленным Telegram она открывает приложение и сразу добавляет MTProto-бридж без ручного ввода.
- **Inline-SVG диаграмма потока пакетов** (`телефон → роутер → ZAPRET CORE PC → DPI → сеть`) с цветной легендой — наглядно показывает, что это **не VPN** и трафик никуда наружу не уходит.
- **Совместимость, FAQ, Troubleshooting** на одной странице: матрица «что работает где», аккордеон из 8 частых вопросов и 6 карточек «если не работает» с конкретными шагами.
- **Tap-to-copy на всём подряд** — IP, порт, PAC URL, любые `<code>` сниппеты в инструкциях; рядом всплывает тост «Скопировано».
- **Безопасный `window.ZC` bootstrap** — вся динамика (IP, порты, секрет, Telegram deep-link, дата сборки) сериализуется один раз в frozen-объект, остальной HTML/JS статичен. Добавить новое поле теперь = одна строка в C++ + один тег `data-bind` в разметке.

### UI: Логи

- Раздел **LOGS** перерисован на утилитарную консоль: счётчики простым текстом, фильтр уровня сегментным переключателем, фильтр сервиса одним комбобоксом. Строки идут фиксированными колонками `TS / LVL / SVC / MSG`, цвет — только у уровня (`INF` серый, `WRN` жёлтый, `ERR` красный). Сервис рендерится нейтральным серым.
- Под капотом большие логи теперь рендерятся через `ImGuiListClipper`, без push-стилей на каждую невидимую строку — отзывчивость заметно выше при тысячах записей.
- В сайдбар добавлен значок для **LOGS** (терминальный `>` с подчёркиванием), раньше эта строка была без иконки.

### UI: Settings

- Удалена подвкладка **«Сервисы»** после «Инструменты». Навигация в SETTINGS снова сжата до **Профили / Тест / Инструменты**. `LoadConfig` корректно подрезает легаси `settings_view=3` из старых конфигов.
- `ServiceRegistry` оставлен на месте как источник данных для путей запуска движка, MTProto, HTTP-прокси и веб-сервера — он используется и за пределами вкладки, удалять его за рамками задачи.

## Архитектура и оптимизация

Проект полностью избавляет от необходимости использовать командные оболочки (CMD/PowerShell) в повседневном использовании.

* **Технологический стек:** Разработано на чистом C++ (DirectX 11 + ImGui). Исполняемый файл требует прав администратора (манифест `requireAdministrator`) для взаимодействия с WinDivert API.
* **Умная синхронизация UI:** Интерфейс общается с ядром Windows. При запуске он сканирует активные службы и автоматически восстанавливает статусы и чекбоксы.
* **Нулевая задержка и высокий приоритет:** Процесс инициализируется с флагом `HIGH_PRIORITY_CLASS`. Нагрузка на ЦП в простое составляет **0%**. Реализован алгоритм `ReadExactly` для защиты от TCP-фрагментации.

## Технические особенности

* **Адаптивная фильтрация (Умные профили):** 7 предустановленных векторов атак DPI (Classic, Aggressive, Mobile, Failsafe) для разных интернет-провайдеров.
* **Фоновая интеграция (Службы Windows):** Бесшовная установка в системные службы. Обход и прокси для Telegram работают невидимо на уровне ОС.
* **VoIP и игровой режим:** Выделенное правило маршрутизации для портов **1024-65535 (TCP/UDP)**. Предотвращает потерю пакетов в играх и исправляет работу Discord.
* **Обход блокировки DNS:** Динамическая подмена заголовков `Host` и `Origin` (SNI Spoofing). Always-on DoH-резолвер на `127.0.0.1:53` через Cloudflare 1.1.1.1.
* **Мобильные устройства:** HTTP CONNECT прокси для Android и iOS с автономной фоновой службой и расширенной Setup Node страницей (6 платформенных вкладок, FAQ, troubleshooting, dual-QR, Telegram deep-link).

## Использование: Ускорение Telegram

**Вариант 1. Локальный запуск (Работает, пока открыто приложение):**

1. Перейдите во вкладку **НАСТРОЙКИ**.
2. Поставьте галочку **Включить TG обход** (запускается на порту 1080).
3. Нажмите **Применить прокси к Telegram**. Подтвердите запрос в открывшемся приложении Telegram.
4. Нажмите **Запустить обход**, теперь можно свернуть приложение и наслаждаться.

**Вариант 2. Автозапуск и фоновый режим (Рекомендуется):**

1. Поставьте галочку **Включить TG обход**.
2. Нажмите **Применить прокси к Telegram**. Подтвердите запрос в открывшемся приложении Telegram.
3. Нажмите **Установить службу**.
4. Программа установит три системные службы. Теперь можно закрыть окно — всё работает автономно.

## Использование: Обход на телефоне

1. ПК и телефон должны быть в одной Wi-Fi сети.
2. Нажмите **Установить службу** (или **Запустить обход** для локального режима).
3. Откройте на телефоне `http://<IP-вашего-ПК>:8081` — попадёте на **Setup Node** страницу с автоматическим определением платформы.
4. Сразу после открытия:
   * **iOS / Android / macOS / Windows** — активная вкладка подберётся по `User-Agent`, останется только пройти 2–3 шага из инструкции.
   * **Telegram** — нажмите кнопку **«Открыть в Telegram»** под QR-кодами (deep-link `tg://proxy?...`); приложение само добавит MTProto-бридж с вашим IP, портом и секретом.
   * **PAC** — отсканируйте PAC-QR или скопируйте URL и вставьте его в **Wi-Fi → HTTP Proxy → Auto** на телефоне; PAC сам решит, какие домены гонять через обход, а какие пускать напрямую.
5. Готово — все заблокированные сайты открываются на телефоне через обход на ПК.

> [!TIP]
> Если что-то не работает — на той же странице есть разделы **Совместимость**, **Частые вопросы** и **Если не работает** с конкретными шагами под каждую платформу.

## Управление списками доменов

Списками маршрутизации управляется прямо из вкладки **СПИСКИ** в интерфейсе программы.

Физически текстовые файлы находятся в папке `lists/`:

- **`list-general.txt`** — для доменов (поддомены включаются автоматически).
- **`list-google.txt`** — для специфических сервисов и исключений.
- **`list-exclude-user.txt`** / **`ipset-exclude-user.txt`** — для исключения доменов/IP из фильтрации.

## Принципы продукта

1. **Производительность:** Нулевое влияние на исходную сетевую задержку.
2. **Изоляция:** Модификация заголовков (SNI, TLS) выполняется исключительно на локальном ПК без сторонних серверов.
3. **Развертывание в один клик:** Превращение сложных консольных параметров в интуитивно понятный графический интерфейс.
4. **Мобильность:** Единая точка обхода для всех устройств в сети — HTTP-прокси и расширенная Setup Node страница с пошаговыми гайдами под каждую платформу.
