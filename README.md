**Альтернатива консольным утилитам. Аппаратно-ускоренный GUI для обхода ограничений DPI.**

Ускорение Telegram Desktop, YouTube, Discord и полная кастомизация.

> **ZAPRET CORE** — это аппаратно-ускоренный графический пользовательский интерфейс (GUI) и система оркестрации для локального обхода систем глубокого анализа пакетов (DPI).
>
> В отличие от классических VPN-сетей, эта утилита не маршрутизирует трафик через сторонние удаленные серверы. Внедрение, фрагментация и модификация пакетов происходят локально на уровне ядра ОС. Такой подход позволяет обходить ограничения провайдера, сохраняя 100% пропускной способности канала и исходный сетевой пинг.

> [!WARNING]
> ### РЕАКЦИЯ АНТИВИРУСОВ И СИСТЕМЫ
> Программа использует драйвер `WinDivert64.sys` для обработки сети на уровне ядра ОС. Некоторые антивирусы склонны классифицировать файлы WinDivert как высокорискованные. Обнаружение всегда будет называться `WinDivert` или `Not-a-virus:RiskTool.Multi.WinDivert`.
>
> Это абсолютно нормальное поведение. WinDivert — это легитимный инструмент для перехвата трафика. Для стабильной работы добавьте папку с утилитой в исключения вашего антивируса.

**ZAPRET CORE: UPDATE v2.5.5 — «The Hardening Pass»**
 
 
**Основные изменения:**
 
**1. HOSTS Manager — управление файлом hosts прямо из приложения**
 
 
> Добавлена новая вкладка **HOSTS** между SETTINGS и LISTS. Менеджер позволяет одним кликом разблокировать сервисы, которые сами режут российские IP (ChatGPT, Spotify, Notion, Claude, Gemini, JetBrains, Adobe, Canva, ElevenLabs, DeepL и десятки других) и которые DPI-обходом не закрываются — это L7-блок на стороне сервиса, а не РКН. Структура UI: «Напрямую из hosts» (тоггл on/off — IP-подмена для WhatsApp, rutor, Discord, x.com, YouTube, GitHub, Discord-voice от Flowseal, Supercell, ntc.party), «ИИ» (per-service DNS-выбор: Откл / Zapret DNS / XBOX DNS / Comss DNS / Malw DNS / Malw DNS v2 / play2go.cloud DNS) и «Остальные» (тот же выбор для TikTok, Twitch, JetBrains, Autodesk и т.д.). Все записи аккуратно складываются в маркированный блок `# ZAPRET CORE HOSTS BEGIN ... END` внутри `C:\Windows\System32\drivers\etc\hosts` — «Очистить» сносит только этот блок, не трогая твои собственные строки. После каждой записи автоматически вызывается `ipconfig /flushdns`, поэтому изменения вступают мгновенно.
 
 
 
**2. UTF-8 для системных сообщений Windows — устранение «кракозябр» в логах**
 
 
> `FormatMessageA` на русской винде возвращает строку в CP1251, а ImGui-лог работает только с UTF-8 — поэтому в строках типа `(code 87: �������� ����� �������)` юзер видел вопросы вместо «Параметр задан неверно». Реализована функция `Win32ErrMsg(code)` через `FormatMessageW` (UTF-16) → `WideCharToMultiByte(CP_UTF8)` → `std::string`. Теперь любые системные exit-коды и win32-ошибки отображаются на корректной кириллице — это касается и pre-flight диагностики, и watchdog-сообщений «engine exited unexpectedly», и любых других мест где код звал `FormatMessage`.
 
 
 
**3. Windows Autostart через Task Scheduler — починка автозапуска elevated EXE**
 
 
> Старая регистрация в `HKCU\Software\Microsoft\Windows\CurrentVersion\Run\ZapretCore` действительно появлялась в Task Manager → Startup, но **Windows молча игнорировал** автозапуск EXE с `requireAdministrator` manifest при логоне: UAC не может показать диалог в момент начала пользовательской сессии, и процесс просто никогда не стартовал. Это классическая проблема для elevated-приложений и единственное надёжное решение — Task Scheduler с `RunLevel=HIGHEST`. Реализовано через `schtasks.exe /Create /TN ZapretCore_Autostart /TR "\"<exe>\"" /SC ONLOGON /RL HIGHEST /F` — задача создаётся из уже elevated процесса, Windows запоминает право повышения и при логоне запускает приложение без UAC-промта. `IsAutostartEnabled()` теперь делает `schtasks /Query`, `RefreshAutostartState()` ушёл в фон (запрос медленный, ~300мс — не блокируем UI). Тоггл во вкладке Tools одновременно сносит legacy `HKCU\Run`-запись (миграция старых юзеров), чтобы не висели две регистрации.
 
 
**4. Adaptive Scanner — фикс пропадающих кнопок управления**
 
 
> До запуска адаптивного автоподбора кнопки «Перезапуск» и «Очистить память» отображались, но после первого вывода диагностики исчезали и переставали кликаться. Причина — кривое условие видимости в layout-расчёте карточки. Кнопки теперь всегда находятся в сцене, привязаны к `GetContentRegionMax().y` относительно нижнего края `ArcChild`-карточки, и не зависят от количества бейджей «Также работают». Это закрывает регрессию из v2.5.0.
 
 
**5 Account Page Redesign — визуальный апгрейд `account.html`**
 
 
> Страница личного кабинета `zapretcore.ru/account.html` полностью переработана визуально — без правок JS-логики, DOM-структуры и эндпоинтов `/api/*`. Атмосферный слой (шум + scanlines + vignette) подключён как на index/donators/games-страницах с авто-инверсией под тёмную/светлую тему. Заголовки `.acc-h` получили 165° градиент с анимированным `borderGlow`, в светлой теме подключается фиолет `--grad-2`. Шапка `.nav` использует `backdrop-filter: blur(40px) saturate(180%)`. Пустой аватар отрисовывается через SVG-камеру в CSS-mask вместо литерала «+». Тосты получили цветной бордер и SVG-иконки (чек/крест/info), кнопки — `:active` spring-squish и `aria-busy` спиннер. `prof-stats` перешёл на 4 колонки с эллипсисом для длинных `@nick`. **A11y**: `:focus-visible` показывает кольцо только при Tab-навигации, `prefers-reduced-motion` отключает spin/float-анимации, `hover:none` гасит hover-эффекты на тач-устройствах, `::selection` использует брендовый цвет, жёлтый автозаполнения убран. **Багфиксы**: `display=isDesktop?'none':'none'` (всегда `'none'` независимо от ветки) → корректная мобильная пагинация; дублирующий `.prof-stats` flex-объявление; `font-size: .44rem` (нечитаемо на high-DPI) → `.5rem`; 3 inline-стиля унесены в CSS-классы.
 
 
 
**6. MTProto WebSocket Handshake — устранение регрессии v2.4.4 → v2.4.3**
 
 
> В версии v2.4.4 при попытке настроить WinHTTP-сессию был добавлен вызов `WinHttpSetOption(..., WINHTTP_OPTION_UPGRADE_TO_WEB_SOCKET, ...)` с указателем на dummy-DWORD. Это ломало WebSocket-handshake — сама операция возвращала ERROR_INVALID_PARAMETER в момент `WinHttpSendRequest`, и MTProto-мост не мог выйти на DC через WS. Откат на `WinHttpSetOption(..., NULL, 0)` (как в v2.4.3) восстановил handshake. В лог теперь корректно прилетает `MTProto: DC2 WS pool hit via 149.154.167.220` — прямой WebSocket к Telegram-DC через пул предоткрытых сессий, моментально, без re-handshake.
 
 
**7. Force DNS — полное удаление модуля**
 
 
> Force DNS, добавленный в v2.5.0, в продакшене вёл себя нестабильно: при отключении тоггла иногда оставлял адаптеры на 8.8.8.8 (вместо отката на DHCP), что ломало MTProto-bridge и DNS-резолв в принципе на машинах с локальным резолвером (NAS, AdGuard Home, Pi-hole). Поскольку DPI-движок и так сам обходит DNS-инспекцию провайдера через DoH-резолвер `StartDohResolver()`, отдельный Force DNS не давал реальной пользы — только риск. Модуль удалён целиком (-174 строки): UI-плашка, тоггл, фоновый watcher, persistence в `config.ini`. Юзеры со специфическими DNS-сценариями могут выставить нужный резолвер сами через свойства адаптера или роутер.

## Архитектура и оптимизация

Проект полностью избавляет от необходимости использовать командные оболочки (CMD/PowerShell) в повседневном использовании.

* **Технологический стек:** Разработано на чистом C++ (DirectX 11 + ImGui). Исполняемый файл требует прав администратора (манифест `requireAdministrator`) для взаимодействия с WinDivert API.
* **Умная синхронизация UI:** Интерфейс общается с ядром Windows. При запуске он сканирует активные службы и автоматически восстанавливает статусы и чекбоксы.
* **Нулевая задержка и высокий приоритет:** Процесс инициализируется с флагом `HIGH_PRIORITY_CLASS`. Нагрузка на ЦП в простое составляет **0%**. Реализован алгоритм `ReadExactly` для защиты от TCP-фрагментации.

## Технические особенности

* **Адаптивная фильтрация (Умные профили):** 7 предустановленных векторов атак DPI (Classic, Aggressive, Mobile, Failsafe) для разных интернет-провайдеров.
* **Фоновая интеграция (Службы Windows):** Бесшовная установка в системные службы. Обход и прокси для Telegram работают невидимо на уровне ОС.
* **VoIP и игровой режим:** Выделенное правило маршрутизации для портов **1024-65535 (TCP/UDP)**. Предотвращает потерю пакетов в играх и исправляет работу Discord.
* **Обход блокировки DNS:** Динамическая подмена заголовков `Host` и `Origin` (SNI Spoofing). Always-on DoH-резолвер на `127.0.0.1:53` через Cloudflare 1.1.1.1.
* **Мобильные устройства:** HTTP CONNECT прокси для Android и iOS с автономной фоновой службой и расширенной Setup Node страницей (6 платформенных вкладок, FAQ, troubleshooting, dual-QR, Telegram deep-link).

## Использование: Ускорение Telegram

**Вариант 1. Локальный запуск (Работает, пока открыто приложение):**

1. Перейдите во вкладку **НАСТРОЙКИ**.
2. Поставьте галочку **Включить TG обход** (запускается на порту 1080).
3. Нажмите **Применить прокси к Telegram**. Подтвердите запрос в открывшемся приложении Telegram.
4. Нажмите **Запустить обход**, теперь можно свернуть приложение и наслаждаться.

**Вариант 2. Автозапуск и фоновый режим (Рекомендуется):**

1. Поставьте галочку **Включить TG обход**.
2. Нажмите **Применить прокси к Telegram**. Подтвердите запрос в открывшемся приложении Telegram.
3. Нажмите **Установить службу**.
4. Программа установит три системные службы. Теперь можно закрыть окно — всё работает автономно.

## Использование: Обход на телефоне

1. ПК и телефон должны быть в одной Wi-Fi сети.
2. Нажмите **Установить службу** (или **Запустить обход** для локального режима).
3. Откройте на телефоне `http://<IP-вашего-ПК>:8081` — попадёте на **Setup Node** страницу с автоматическим определением платформы.
4. Сразу после открытия:
   * **iOS / Android / macOS / Windows** — активная вкладка подберётся по `User-Agent`, останется только пройти 2–3 шага из инструкции.
   * **Telegram** — нажмите кнопку **«Открыть в Telegram»** под QR-кодами (deep-link `tg://proxy?...`); приложение само добавит MTProto-бридж с вашим IP, портом и секретом.
   * **PAC** — отсканируйте PAC-QR или скопируйте URL и вставьте его в **Wi-Fi → HTTP Proxy → Auto** на телефоне; PAC сам решит, какие домены гонять через обход, а какие пускать напрямую.
5. Готово — все заблокированные сайты открываются на телефоне через обход на ПК.

> [!TIP]
> Если что-то не работает — на той же странице есть разделы **Совместимость**, **Частые вопросы** и **Если не работает** с конкретными шагами под каждую платформу.

## Управление списками доменов

Списками маршрутизации управляется прямо из вкладки **СПИСКИ** в интерфейсе программы.

Физически текстовые файлы находятся в папке `lists/`:

- **`list-general.txt`** — для доменов (поддомены включаются автоматически).
- **`list-google.txt`** — для специфических сервисов и исключений.
- **`list-exclude-user.txt`** / **`ipset-exclude-user.txt`** — для исключения доменов/IP из фильтрации.

## Принципы продукта

1. **Производительность:** Нулевое влияние на исходную сетевую задержку.
2. **Изоляция:** Модификация заголовков (SNI, TLS) выполняется исключительно на локальном ПК без сторонних серверов.
3. **Развертывание в один клик:** Превращение сложных консольных параметров в интуитивно понятный графический интерфейс.
4. **Мобильность:** Единая точка обхода для всех устройств в сети — HTTP-прокси и расширенная Setup Node страница с пошаговыми гайдами под каждую платформу.
